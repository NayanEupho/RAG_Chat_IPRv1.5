## 1. Scope of the Work

The Scope of the Work Includes:
1. Design, development, testing and deployment of "Institute for Plasma Research" (IPR) website to be hosted on a server provided at IPR.
2. Build a responsive, user-friendly design, bilingual (English & Hindi) website with custom translation.
3. Adhere to accessibility standards for persons with disabilities (PwDs).
4. Migrate content from existing website to the newly developed website.
5. Implement SSL/TLS encryption, obtain "Safe-to-Host" certificate, "Website Quality Certificate" and conduct a "Web Accessibility Audit".
6. Provide training and technical manuals for website management and troubleshooting.
7. One Year functional Warranty to be provided after acceptance.
8. The work mentioned above shall be completed within 165 Days from the date of award of order/contract.

### 1.1. Web Application

The website will be developed to include the following web application features:
1. Design, develop, test and deploy the website to comply fully with the Guidelines for Indian Government Websites and Apps GIGW 3.0, W3C and WCAG 2.1 'AA' standards/Guidelines.
2. Ensure the website supports dynamic page expansion as needed.
3. Optimize page load times using compressed images, minified CSS/JS and lazy loading.
4. Implement a search field with autocomplete, fuzzy matching and ranked results based on indexed content.
5. Prohibit third-party tracking scripts or hidden backlinks.
6. Maintain separate production and development environments.
7. Establish a defined content workflow encompassing creation, review, approval, and publication phases.
8. Enable scheduled publishing/unpublishing of web pages by date/time.
9. Implement a dashboard for managing content (create, delete, update, and upload).
10. Display pending tasks on dashboard with one-click navigation and enable notifications.
11. Implement a modular architecture to support future integrations without disrupting functionality.
12. Configure RBAC (Role-Based Access Control) for hierarchical user roles.
13. Include default image and video gallery templates with administrative controls for content management.
14. Create/manage info graphics, images, videos and audio files based on raw content provided by IPR.
15. Fully optimized website for Search Engine Optimization (SEO) including metadata, responsive design, fast load time, secure URLs, quality content, crawlability, structured data and search console integration.
16. Site Analytics may contain the following but not limited to: Number of users on website, Hit rate and source domains/IP address, List of referrers, Page wise hit rate.
17. Ensure the application is scalable and layered to allow the addition of new schemes/modules/user application forms.
18. Report generation based on website content modifications, with the ability to filter results by a specified timeline.

### 1.2. UI/UX Design

The website should be developed to ensure usability, accessibility, and engagement with the following designs:
1. Design responsive layouts that adapt automatically to different screen resolutions.
2. Deliver a website compatible with all major browsers, tablets and mobile devices across Windows, Android and Mac iOS platforms.
3. Implement responsive design that adapts to both landscape and portrait orientations.
4. Optimize content visibility and navigation based on device screen characteristics.
5. Ensure full accessibility for persons with disabilities (PwDs).

### 1.3. Security

The following measures will be implemented to ensure security and protect against common vulnerabilities:
1. Implement form-input validation on client/server side.
2. Protect login/sign-in and user inputs with CAPTCHA.
3. Enforce HTTPS using a verified SSL certificate.
4. Separate application and data storage paths and hide actual file paths.
5. Define all web paths using routes/XML and return HTTP 404 for undefined paths.
6. Prevent SQL injection, Cross Site Scripting (XSS), Cross Site Request Forgery (CSRF), DOS/DDoS and hacking/defacing attempts.
7. Guard against brute force attacks on login page.
8. Lock accounts after repeated failed authentication attempts.
9. Exclude sensitive data from cookies.
10. Implement proper session management to avoid session replay problems.
11. Implement multi-level error handling (client, middle-tier, backend).
12. Deploy an Open-Source Web Application Firewall (WAF) to prevent cyberattacks.
13. Implement searchable and achievable audit trails for all actions.
14. Scan files for malware/viruses before uploading.
15. Enforce integrity checks and block unauthorized binary execution.
16. Restrict access to the administrator panel.
17. Host the website on the latest OS with restricted network access.
18. Implement automatic regular backups of the website to ensure quick recovery in case of failure.

### 1.4. Certificates and Audit

The vendor is responsible for obtaining the following certificates/audit report:
1. "Safe-to-host" certificate issued by the cybersecurity auditors empanelled by CERT-In/STQC or the auditors of STQC or NIC.
2. Website Quality Certificate (WQC) issues by Standardisation Testing and Quality Certification (STQC).
3. Web Accessibility Audit by empanelled Web Accessibility auditors by Department of Empowerment of Persons with Disabilities, Ministry of Social Justice & Empowerment, Govt. Of India.

### 1.5. Requirement for Work Execution

The following standards and processes will govern project implementation:
1. Develop and submit three interactive design themes for IPR's approval.
2. Maintain continuous engagement with the IPR team during the development of the website. Conduct weekly/fortnightly review meetings and document all project updates.
3. IPR retains full ownership of all hosted data and source code/software used for the development of website.
4. IPR will provide In-house hosting with the latest operating system, SSL certificates and DNS server for domain hosting.
5. Submit a duly signed Non-Disclosure Agreement (NDA) undertaking to protect all confidential project information (Appendix-1) after placement of Purchase Order as per IPR format attached in Buyer Added Bid Specific ATC.

## 2. Warranty Requirement

The vendor is required to perform the following activities:
1. Provide technical support for 1 year against any issue faced in website functioning (management and troubleshooting).
2. Provide a dedicated single point of contact during the period of the contract.
3. Removal of bugs/ Rectification of errors in website etc.
4. Restore the website and all supportive software in case of server/software crashes.
5. Website patch management, patch updates and upgrades.
6. Maintain the integrity of the website against spam, ransomware, hackers and viruses.

## 3. Acceptance and Deliverables

Following are the deliverables of the Contract:
1. Document on "Software Requirement Specification (SRS)" approved by IPR.
2. Design documents based on the approved version of SRS.
3. Report on Test Cases and User Acceptance Testing (UAT).
4. Newly developed website deployment and Go-live at IPR.
5. Handing over source code and credentials for developed website.
6. Training and Technical manuals for website management and troubleshooting.
7. Safe-to-host Certificate, Website Quality Certificate (WQC) and Web Accessibility Audit report.

Final acceptance will be given only after completion of work and submission of above (Sr. 1 to 7) deliverables.
